Ecosystem · Category 1

1Built and run by HomeTest

What HomeTest builds, runs and staffs directly

The HomeTest BAU team sizes around this. Ten elements make up the core HomeTest owns end-to-end. Everything else in the ecosystem sits outside the team.

Integration spine In flight
The orchestration layer that routes ordering, dispatch, results and status across the ecosystem. Status: live for HIV; scaling for PSA and HbA1c configurations as they enter scope.
Framework instrument In flight
Commercial vehicle and call-off architecture. Admits, contracts and exits suppliers. Status: PA23 settled as the procurement regime. Within-PA23 vehicle choice (Open Framework, Dynamic Market, DPC) still in flight.
Pattern catalogue In flight
Library of reusable overlay options across initiation, commissioning, supply chain and the rest. Status: live for HIV patterns; PSA and HbA1c patterns being added; pharmacy-init pattern subject to an open SLT scope decision.
Test catalog In flight
The assurance and discoverability layer: which tests, which providers, which pathways, with the SNOMED codes that make them addressable in the wider NHS data estate. The catalog is what lets commissioners and clinicians find a test and trust it has been admitted. It is not a marketplace. There is no purchase or transact function inside it; buying happens through the framework. Status: principles to be ratified at programme level. Gap analysis against the FDP genomics catalog outstanding.
Configurations In flight
Live deployments of specific pattern combinations: HIV via LA, PSA via acute trust, HbA1c via GP. Status: HIV configuration live in private beta; PSA configuration in design; HbA1c configuration in discovery.
IG framework structure In flight
Federated multi-controller model and the artefact stack (DPIA, DSA, DPA, IAR) every configuration inherits. Status: federated multi-controller model agreed in principle. Lawful basis position open at SLT level. IAO appointment is the critical blocker.
Clinical safety case structure
Hazard log structure, SLSP template, configuration sign-off route.
Hazard log structure
The structure into which configuration-specific hazards are logged and reviewed.
Onboarding gateway In flight
The seven-stage supplier journey from EOI to admission. Status: in operation for Alpha suppliers; Beta evolution adds the public market engagement and full seven-stage flow.
MI and data flow integrity In flight
Platform-level reporting, anomaly detection, access controls, audit logs. Status: live for HIV; KPI framework not yet designed; outcome attribution methodology undefined.
The operating model itself In flight
Stewardship of the five frames, the cast, the lifecycle, the governance shape. Evolved by the Commercial Lead function. Status: current discovery. Twelve-week design period concluding mid-June with a recommendation paper to SLT.
